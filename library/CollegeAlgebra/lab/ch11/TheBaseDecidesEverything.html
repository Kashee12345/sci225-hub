The Base Decides Everything

An exponential function is y = a·bx, and almost all of its personality lives in that one number b. Above 1 the curve climbs; below 1 it falls; at exactly 1 it stops being exponential at all. Why the base runs the whole show

A linear function adds the same amount at every step. An exponential function multiplies by the same amount at every step. That one swap — add becomes multiply — is the entire difference, and the number being multiplied in is the base.

The defining property: the ratio is always b

Take any exponential y = a·bx and compare one output to the one before it. Divide:

the step ratio f(x + 1)f(x) = a·bx+1a·bx = b

Do not take that on faith — watch the factors cancel. Nothing here is a rule, it is just counting how many b's are on top and on the bottom:

count the factors b5 = b·b·b·b·b   five of them
b4 = b·b·b·b   four of them
b5b4 = b·b·b·b·bb·b·b·b = b   four pairs cancel, exactly one b survives

The exponent x does not have to be a whole number for this to hold, but whole numbers are where you can literally see it. And notice the a cancelled too — the starting value has no say in the step ratio. Whatever a is, the jump from each output to the next is a multiplication by b, and only by b. Why b > 1 must grow and 0 < b < 1 must decay

This is now a two-line argument rather than something to memorize. Every output a·bx is a positive number (proved below), and the next output is that positive number times b. So ask the only question that matters: does multiplying a positive number by b make it bigger or smaller?

if b > 1 f(x + 1) = f(x)·b > f(x)·1 = f(x)  →  every output beats the one before — growth
if 0 < b < 1 f(x + 1) = f(x)·b < f(x)·1 = f(x)  →  every output falls short of the one before — decay
if b = 1 f(x + 1) = f(x)·1 = f(x)  →  nothing ever changes — a horizontal line

Multiplying an inequality by the positive number f(x) is what lets each line go through, and that is legal precisely because outputs are never zero or negative. So the sorting rule is not arbitrary: compare b to 1 and you have already decided the shape of the graph. Nothing about a enters into it.

The gap this opens is not a small one

Start two sequences at 2. One adds 3 each step; the other multiplies by 3 each step. For a moment they look comparable:

At step 1 the two are 5 and 6 — practically a tie. Four steps later one is at 14 and the other at 162. Push it further and the linear side never recovers: at step 10 adding gives 32, multiplying gives 118,098. This is why the base is worth reading carefully before anything else.

Why the outputs are never zero and never negative

The base is required to be positive, so for a whole number n, bn is a product of positive numbers and is positive. Negative exponents are the reciprocals of those, so they are positive too:

where negative exponents come from b0bn = b0−n = b−n,  and also  b0bn = 1bn  →  b−n = 1bn
a positive number divided into 1 is still positive — and a fraction with numerator 1 can never equal 0

So bx > 0 for every real x, and with a > 0 the whole output stays above the x-axis forever. As x marches toward −∞ on a growth curve, the outputs keep halving, or thirding, or whatever b dictates — they shrink without limit but never arrive at 0. That is what a horizontal asymptote is: a line the graph closes in on and never reaches. For y = a·bx that line is always y = 0, the x-axis itself. A decay curve does the same thing on the right-hand side instead.

One consequence worth stating outright: an exponential of this form has no x-intercept. Setting a·bx = 0 forces bx = 0, which nothing satisfies. Do not go hunting for one.

Every one of them passes through (0, a)

Put x = 0 into y = a·bx and you get y = a·b0 = a·1 = a. The reason b0 = 1 is the same cancelling as before, read in one extra direction:

b3b3 = 1   anything nonzero over itself   and   b3b3 = b3−3 = b0  →  b0 = 1

So the base changes how steeply the curve leaves the point (0, a), and which direction it leaves in, but it cannot move that point. Drag b across its whole range above and watch the gold dot sit perfectly still. Only a lifts it. That also gives you a free reading in reverse: whatever height a graph crosses the y-axis at, that height is a.

Every decay curve is a growth curve in a mirror

Bases b and 1/b are not two separate stories. Watch:

reciprocal base (1/b)x = (b−1)x = b−x   a power of a power multiplies the exponents

Replacing x with −x is exactly what reflects a graph across the y-axis, because the height that used to sit at x = 3 now sits at x = −3. The table makes it plain — the second row is the first row read backwards:

Turning a percent into a base

Most real exponentials arrive described as a percent per period, not as a base. The conversion is one line, and it comes from the fact that keeping what you had and adding r of it means multiplying by 1 + r:

the conversion new = old + r·old = old·(1 + r)  →  b = 1 + r   with r written as a decimal, negative for a loss
described asrb = 1 + rbehavior
up 8% per year+0.081.08growth
up 100% per year+1.002doubling
down 8% per year−0.080.92decay
down 50% per year−0.500.5halving

Two traps live in that table. First, a loss of 8% is not the reciprocal of a gain of 8%. Check: 1.08 × 0.92 = 0.9936, not 1. Gain 8% then lose 8% and you are down to 99.36% of where you started — the loss is taken on a bigger number than the gain was. Second, a 100% increase means b = 2, not b = 1. The base counts what you end with, not what you added.

Two bases are outlawed, and one more is a trap

b ≠ 1. With b = 1, y = a·1x = a for every single x, because 1 multiplied by itself any number of times is still 1. The graph is a horizontal line. There is no growth, no decay, no asymptote to approach — it is a constant function wearing an exponential costume, so it is excluded by definition. Drag the base slider to exactly 1 and watch every reading above go flat.

b > 0. Zero fails first: y = 0x gives 0 at x = 3, but at x = −3 it demands 1 ÷ 03 = 1 ÷ 0, which is undefined, and 00 is undefined as well. Negative bases fail differently and more interestingly. Try b = −4:

why a negative base cannot be a curve (−4)1 = −4,  (−4)2 = 16,  (−4)3 = −64   the sign flips every step — the points hop up and down
(−4)1/2 = √−4  →  not a real number   and the same failure at every exponent with an even denominator

Some negative-base powers do exist — (−8)1/3 = −2, since odd roots of negatives are fine. That is what makes this worse, not better: the legal inputs would be scattered among illegal ones with gaps everywhere between them, so there is no continuous curve to draw at all. Requiring b > 0 is what buys you a smooth graph over every real x.

And a ≠ 0 — the trap. If a = 0 then y = 0·bx = 0 everywhere, flattening onto the x-axis no matter how dramatic the base is. A negative a is perfectly legal, incidentally: it reflects the whole curve below the x-axis, so the range becomes y < 0 while the asymptote y = 0 and the growth-or-decay verdict from b stay exactly as they were.

Domain, range, and what to state on an answer

Domain: all real numbers. With b > 0 there is nothing to exclude — no denominator holding x, no even root of a negative. Write −∞ < x < ∞. This is different from the rational functions of the last chapter, where excluded values were the first thing to hunt for.

Range: y > 0 when a > 0, and y < 0 when a < 0. Never “all real numbers,” and never including 0. Asymptote: y = 0 always, for this form.

On extraneous solutions. They do not arise from the graphing here, but they will the moment you start solving. Two habits protect you. When you solve bx = k, a solution exists only if k > 0 — asking 3x = −5 or 3x = 0 has no answer at all, since the left side is stuck above zero. And if an exponential equation was reached by squaring, or by clearing a denominator, substitute your answer back into the original statement before writing it down, because those moves can invent roots that were never legal. The graph shows you every value the function actually takes; the original problem decides which ones you are allowed to report.

Reading an exponential, in order.  1. Find the base — the number carrying x on its shoulder, not the number in front.  2. Compare it to 1: bigger is growth, between 0 and 1 is decay, exactly 1 is not exponential.  3. Read a; the graph goes through (0, a) whatever the base does.  4. Convert to a percent with r = b − 1 if the problem is worded in percents.  5. Write the asymptote y = 0 and the range — they are free marks people forget.
Six ways this goes wrong.  • Reading 2x as 2x. At x = 5 those are 32 and 10; at x = 10 they are 1024 and 20.  • Confusing x2 with 2x. The variable is in the base of one and the exponent of the other. They happen to agree at x = 2 and x = 4 (both give 4 and 16), and x2 is briefly ahead between them — at x = 3, 9 beats 8 — but past x = 4 the exponential leaves it behind permanently.  • Believing decay reaches zero. It halves forever; forever is not the same as arriving.  • Taking a percent loss off the original amount every period. That is linear. Exponential decay takes the percent off whatever is left.  • Writing b = 1.25 for a 25% decrease. A decrease means b = 1 − 0.25 = 0.75.  • Dropping parentheses: −2x means −(2x), which is negative, while (−2)x is not a legal exponential at all.
