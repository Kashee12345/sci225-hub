Check the Domain

Somewhere in the middle of almost every logarithmic equation there is a step that widens what the expression will accept. Combine log x + log(x − 3) into log(x(x − 3)) and negative x is suddenly allowed. The algebra after that point is honest — but it is answering a slightly bigger question than the one you were asked, and it will hand back an extra answer that must be rejected. The domain check

A negative candidate is not automatically wrong

log(x2) = 2  →  x2 = 102 = 100  →  x = ±10
x = 10  →  argument x2 = 100✓ keep
x = −10  →  argument x2 = 100✓ keep
Both work: log(100) = 2 either way. The test is never “is x negative?” — it is “is every argument positive?” Rejecting negatives on sight is a habit, not a rule, and it will cost you a real solution.
The whole discipline in three lines: (1) before you solve, write down what each argument requires; (2) solve; (3) test every candidate in the original equation, not in the line you rewrote. Step 3 is the one people skip, and it is the only one that catches the impostor.
