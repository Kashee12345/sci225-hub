One Factor Must Be Zero

Most equations make you chase the variable across the page. A quadratic set against zero does something stranger and much kinder: once you write it as a product, the equation falls apart into two tiny problems, and you solve each one in a single step. That shortcut is the zero product property, and it works for one reason only — zero is the single number that cannot be built out of nonzero pieces. Why zero — and only zero

The property. If A·B = 0, then A = 0 or B = 0 (possibly both).

The proof is one line. Suppose A·B = 0 and suppose A ≠ 0. Because A is not zero, you are allowed to divide both sides by it: B = 0 ÷ A = 0. So the moment one factor refuses to be zero, the other one has no choice. That is the entire mechanism — and notice it leans on A ≠ 0, which is exactly why dividing by a variable is dangerous elsewhere.

Why it fails for any other number. Suppose instead that A·B = 12. Now you have learned nothing about A or B by themselves, because 12 can be assembled a thousand ways: 3·4, 2·6, 1·12, (−3)·(−4), 24·½, and on forever. Zero is the one product that cannot be faked. To land on zero, a factor must already be zero. The uniqueness of that recipe is the whole reason the shortcut exists.

The trap. The property says nothing at all unless the other side is zero. From (x − 1)(x + 2) = 4 you may not write x − 1 = 4. That would give x = 5, and (5 − 1)(5 + 2) = 4 · 7 = 28, not 4. Move everything to one side and factor first, every single time.
The routine. 1. Get 0 alone on one side.  2. Factor the other side into a product.  3. Set each factor equal to zero.  4. Solve those small equations.  5. Check each answer in the original. Unlike squaring both sides, this method invents no extraneous solutions — but the check still catches an arithmetic slip, so keep it.
